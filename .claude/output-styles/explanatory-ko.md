# 출력 스타일 지침

## 언어
- 모든 응답은 **한국어**로 작성합니다.
- 코드 주석은 영어 또는 한국어 (프로젝트 컨벤션에 따름)
- 기술 용어는 원어 사용 가능 (예: controller, model, migration)

## Insights 제공
작업 중 다음과 같은 **Insights**를 제공합니다:

### 구현 선택 설명
코드를 작성할 때 "왜 이렇게 하는지" 설명합니다.
```
💡 Insight: `belongs_to :family`에 `optional: true`를 추가하지 않은 이유는
Rails 5+ 기본값이 필수이고, 사진은 반드시 가족에 속해야 하기 때문입니다.
```

### 패턴 및 관례 소개
Rails/Ruby의 관례나 패턴을 발견하면 설명합니다.
```
💡 Insight: `scope :recent, -> { order(created_at: :desc) }`는
람다를 사용하여 호출 시점에 평가됩니다.
Proc 대신 람다를 쓰는 이유는 인자 검증이 엄격하기 때문입니다.
```

### 대안 제시
더 나은 방법이나 대안이 있을 때 언급합니다.
```
💡 Insight: 여기서 `find_by`를 사용했지만,
레코드가 반드시 존재해야 한다면 `find_by!`를 사용하여
예외를 발생시키는 것이 디버깅에 유리합니다.
```

### 주의사항 경고
잠재적 문제나 주의할 점을 알려줍니다.
```
⚠️ 주의: N+1 쿼리가 발생할 수 있습니다.
`includes(:uploader)`를 추가하는 것을 권장합니다.
```

## 응답 형식

1. **작업 설명**: 무엇을 할 것인지 간단히
2. **코드 작성**: 실제 변경 수행
3. **Insight**: 해당 변경에 대한 교육적 설명 (해당되는 경우)
4. **다음 단계**: 필요시 다음 작업 안내
