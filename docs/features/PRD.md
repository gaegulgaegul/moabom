## 1. 제품 개요

### 1.1 제품명
**모아봄 (MoaBom)**
- 의미: "모아서 봄" + "봄(Spring)처럼 새 생명의 시작"
- 영문: MoaBom

### 1.2 제품 비전
> "아이의 성장을 가족과 함께 기록하고, 언제든 다시 봄처럼 따뜻하게 돌아볼 수 있는 공간"

### 1.3 미션
바쁜 부모들이 **쉽고 빠르게** 아이 사진을 정리하고, **안전하게** 가족과 공유하며, **소중한 순간**을 영원히 간직할 수 있도록 돕는다.

### 1.4 핵심 가치 제안 (Value Proposition)

| 가치 | 설명 |
|-----|------|
| **간편함** | 3탭 이내로 사진 업로드 및 공유 완료 |
| **자동화** | AI가 날짜/얼굴/상황별로 자동 정리 |
| **프라이버시** | 가족만 볼 수 있는 비공개 공간 |
| **추억 회상** | "N년 전 오늘" 알림으로 추억 되새김 |

## 2. 타겟 사용자

### 2.1 주요 페르소나

#### 페르소나 1: 워킹맘 지영 (32세)
- **상황**: 맞벌이, 12개월 아기
- **니즈**: 시간 없이 빠른 정리, 친정/시댁에 공유
- **페인포인트**:
  - 핸드폰에 사진 3,000장 쌓임
  - 매번 카톡으로 보내기 번거로움
  - 좋은 사진 골라낼 시간 없음
- **기대**: "업로드만 하면 알아서 정리되고 부모님이 보셨으면"

#### 페르소나 2: 전업맘 수진 (29세)
- **상황**: 전업주부, 24개월 아기
- **니즈**: 예쁘게 꾸미기, 성장 기록, SNS 공유
- **페인포인트**:
  - 인스타 올리기엔 프라이버시 걱정
  - 성장 앨범 만들 시간 부족
- **기대**: "예쁜 프레임으로 꾸미고 가족만 보게"

#### 페르소나 3: 할머니 영숙 (58세)
- **상황**: 손주 사진 받아보기
- **니즈**: 쉬운 조작, 알림 받기
- **페인포인트**:
  - 카톡 사진 저장하면 용량 부족
  - 예전 사진 찾기 어려움
- **기대**: "새 사진 올라오면 알려주고 쉽게 보기"

### 2.2 타겟 시장

| 구분 | 내용 |
|-----|------|
| **1차 타겟** | 0-5세 자녀를 둔 부모 (2535 세대) |
| **2차 타겟** | 조부모 (5060 세대) |
| **시장 규모** | 한국 영유아 가구 약 200만 |

## 3. 계정 및 사용자 관리

### 3.1 인증 (Authentication)

#### 회원가입/로그인 방식

| 방식 | 우선순위 | 비고 |
|-----|---------|------|
| **카카오 로그인** | Must | 한국 사용자 주력 |
| **Apple 로그인** | Must | iOS 필수 요구사항 |
| **Google 로그인** | Should | Android 편의성 |
| **이메일/비밀번호** | Could | 소셜 로그인 거부 사용자용 |

#### 인증 플로우

```
앱 실행 → 로그인 화면
  ├─ 소셜 로그인 선택 → OAuth 인증 → 토큰 발급 → 메인
  ├─ 신규 가입 → 프로필 생성 → 아이 등록 → 메인
  └─ 자동 로그인 (토큰 유효) → 메인
```

#### 세션 관리

| 항목 | 정책 |
|-----|------|
| Access Token | 1시간 만료 |
| Refresh Token | 30일 만료 |
| 자동 로그인 | 기본 활성화 |
| 동시 로그인 | 최대 5기기 허용 |
| 강제 로그아웃 | 비밀번호 변경 시 전체 로그아웃 |

### 3.2 사용자 프로필

#### 프로필 정보

| 필드 | 필수 | 수정 가능 | 비고 |
|-----|------|----------|------|
| 프로필 사진 | X | O | 기본 아바타 제공 |
| 닉네임 | O | O | 2-20자, 중복 허용 |
| 이메일 | O | X | 소셜 로그인에서 자동 |
| 연락처 | X | O | 알림용 (선택) |
| 가입일 | - | X | 자동 기록 |

#### 프로필 화면 구성

```
[프로필 사진]
닉네임
이메일 (마스킹: ab***@gmail.com)
─────────────────
내 아이 관리 →
가족 그룹 관리 →
─────────────────
알림 설정 →
개인정보 설정 →
앱 설정 →
─────────────────
고객센터 →
로그아웃
계정 삭제
```

### 3.3 아이 프로필 관리

#### 아이 정보

| 필드 | 필수 | 비고 |
|-----|------|------|
| 이름/애칭 | O | 실명 또는 애칭 |
| 생년월일 | O | 개월수 자동 계산 |
| 성별 | X | 테마 추천용 |
| 프로필 사진 | X | 대표 사진 설정 |
| 관계 | O | 엄마/아빠/조부모/기타 |

#### 다자녀 지원

- 최대 10명의 아이 등록 가능
- 아이별 개별 앨범 자동 생성
- 아이 선택 필터 (타임라인에서)

#### 아이 관리 플로우

```
내 아이 관리
  ├─ [+ 아이 추가]
  ├─ 첫째 (3세) [수정] [삭제]
  └─ 둘째 (1세) [수정] [삭제]
```

### 3.4 가족 그룹 관리

#### 역할 및 권한

| 역할 | 권한 | 설명 |
|-----|------|------|
| **소유자** | 모든 권한 | 그룹 생성자, 1명만 존재 |
| **관리자** | 업로드, 삭제, 초대 | 부모 역할 |
| **멤버** | 업로드, 반응, 댓글 | 적극적 참여자 |
| **뷰어** | 보기, 반응만 | 조부모 등 열람 위주 |

#### 가족 초대 방식

| 방식 | 설명 |
|-----|------|
| **카카오톡 공유** | 초대 링크 카카오톡 전송 |
| **링크 복사** | 초대 링크 직접 공유 |

#### 초대 링크 정책

| 항목 | 정책 |
|-----|------|

| 승인 방식 | 자동 승인 (관리자 설정 가능) |

#### 가족 관리 플로우

  ├─ 가족 초대하기 [카카오 | 링크 | QR]

## 4. 핵심 기능 (Features)

### 4.1 MVP 기능 (Phase 1)

#### F1. 사진/영상 업로드 (무제한 대량 업로드)
- 갤러리에서 **무제한 다중 선택** (1000장도 OK)
- 카메라 직접 촬영 → 업로드
- 자동 날짜 인식 (EXIF 메타데이터)
- 중복 사진 감지 및 경고
- **순차 업로드**: 3~5개씩 병렬 처리
- **Direct Upload**: S3 직접 업로드 (서버 부하 없음)
- **진행률 UI**: "150/500 업로드 중..."
- **실패 자동 재시도**: 최대 3회
- **백그라운드 업로드**: 앱 종료해도 계속 (Native)

#### F2. 타임라인 뷰
- 날짜별 자동 정리
- 월별/연도별 그룹핑
- 무한 스크롤
- 핀치 줌 (그리드 크기 조절)

#### F3. 가족 앨범 공유
- 이메일/링크로 가족 초대
- 역할: 관리자 / 멤버 / 뷰어
- 초대된 사람만 접근 가능
- 최대 10명 가족 구성원

#### F4. 반응 및 댓글
- 이모지 반응 (❤️ 😍 😂 🥺 👏)
- 텍스트 댓글
- 푸시 알림 (새 반응/댓글)

#### F5. 클라우드 백업
- 원본 품질 저장
- 자동 백업 (WiFi 시)
- 기기 변경 시 복원

### 4.2 Phase 2 기능

#### F6. 얼굴 인식 자동 태깅
프로필 사진 기반 얼굴 매칭으로 자동 태깅 및 필터링

**기준 얼굴 등록:**
- 아이/가족 프로필 사진을 기준 얼굴로 사용
- 얼굴 → 128차원 벡터(Embedding)로 변환하여 저장
- 프로필 사진 변경 시 기준 벡터 업데이트

**자동 태깅 프로세스:**
```
사진 업로드 → 얼굴 감지 → Embedding 생성 → 기존 프로필과 비교 → 자동 태깅
```

**필터링 기능:**
- "첫째 사진만 보기"
- "할머니랑 찍은 사진"
- "온 가족이 함께한 사진"

**기술 구현 (단계별):**
| 단계 | 방식 | 비용 | 정확도 |
|-----|------|------|--------|
| MVP | 수동 태깅 | $0 | 100% |
| v1.1 | face-api.js (클라이언트) | $0 | ~95% |
| v2.0 | InsightFace/Rekognition | $1/1000장 | ~99% |

**프라이버시 고려:**
- 얼굴 벡터만 서버 저장 (원본 얼굴 이미지 X)
- 클라이언트에서 Embedding 생성 후 전송
- Apple 방식 참고 (온디바이스 우선)

#### F7. AI 스마트 정리
- 베스트샷 자동 선별
- 흐린/중복 사진 자동 분류
- 장면 태깅 (목욕, 식사, 외출 등)

#### F8. 성장 기록
- 키/몸무게 입력
- 성장 그래프 시각화
- 이정표 기록 (첫 걸음, 첫 이앓이 등)
- 개월수 자동 계산

#### F9. 추억 알림
- "N년 전 오늘" 푸시 알림
- 주간/월간 하이라이트
- 기념일 리마인더

#### F10. 테마 프레임
- 월령별 프레임 (100일, 돌 등)
- 시즌 테마 (봄, 크리스마스 등)
- 사진 꾸미기 에디터

### 4.3 Phase 3 기능

#### F11. 하이라이트 영상
- AI 자동 슬라이드쇼 생성
- BGM 추가
- 편집 기능
- 다운로드/공유

#### F12. 포토북 주문
- 앨범 → 실물 포토북 제작
- 다양한 사이즈/디자인
- 배송 연동

#### F13. 고급 프라이버시
- E2E 암호화 옵션
- 스크린샷 감지/차단
- 워터마크 옵션

## 5. 기능 우선순위

### MoSCoW 분석

| 우선순위 | 기능 | 이유 |
|---------|------|------|
| **Must** | 소셜 로그인 (카카오/애플) | 진입 장벽 최소화 |
| **Must** | 사용자 프로필 | 기본 식별 |
| **Must** | 아이 프로필 | 핵심 기능 |
| **Must** | 가족 초대/권한 | 핵심 차별점 |
| **Must** | 무제한 대량 업로드 | 핵심 기능, 사용 편의성 |
| **Must** | 순차 업로드 + 진행률 UI | 대량 업로드 필수 |
| **Must** | 타임라인 뷰 | 핵심 기능 |
| **Must** | 클라우드 백업 | 사용자 신뢰 |
| **Should** | Google 로그인 | Android 편의성 |
| **Should** | 앱 잠금 (생체인증) | 보안 |
| **Should** | 반응/댓글 | 가족 소통 |
| **Should** | 수동 얼굴 태깅 | 필터링 기반 |
| **Should** | 인물별 필터링 | 사용 편의성 |
| **Should** | 성장 기록 | 부가 가치 |
| **Should** | 추억 알림 | 리텐션 |
| **Should** | Native 백그라운드 업로드 | 대량 업로드 UX |
| **Could** | 자동 얼굴 인식 (face-api.js) | 편의성, Phase 2 |
| **Could** | AI 스마트 정리 | 편의성 |
| **Could** | 이메일/비밀번호 로그인 | 대안 제공 |
| **Could** | 2단계 인증 | 고급 보안 |
| **Could** | 테마 프레임 | 감성 |
| **Won't (MVP)** | 고급 얼굴 인식 (Rekognition) | 비용 발생 |

## 6. 사용자 플로우

### 6.1 온보딩 플로우

```
앱 실행
  ↓
소셜 로그인 (카카오/애플/구글)
  ↓
이용약관 및 개인정보처리방침 동의
  ↓
사용자 프로필 생성 (닉네임)
  ↓
아이 프로필 생성 (이름, 생년월일, 관계)
  ↓
갤러리 접근 권한 요청
  ↓
알림 권한 요청
  ↓
첫 사진 업로드 유도
  ↓
가족 초대 안내
  ↓
메인 화면
```

### 6.2 계정 관련 사용자 여정

#### 여정 A: 로그인/로그아웃
```
앱 실행 → 로그인 화면 → 소셜 로그인 선택 → 인증 → 메인 화면
설정 → 로그아웃 → 확인 → 로그인 화면
```

#### 여정 B: 가족 초대 (초대하는 사람)
```
설정 → 가족 관리 → 초대하기 → 방식 선택 (카카오/링크/QR) → 공유 → 완료
```

#### 여정 C: 가족 참여 (초대받은 사람)
```
초대 링크 클릭 → 앱 설치 (미설치 시) → 소셜 로그인 → 가족 참여 확인 → 메인
```

#### 여정 D: 프로필 수정
```
설정 → 프로필 → 수정할 항목 선택 → 수정 → 저장
```

#### 여정 E: 아이 추가
```
설정 → 내 아이 관리 → 아이 추가 → 정보 입력 → 저장 → 앨범 자동 생성
```

#### 여정 F: 계정 삭제
```
설정 → 계정 삭제 → 안내 확인 → 본인 인증 → 7일 유예 안내 → 삭제 예약
```

### 6.3 핵심 사용자 여정

#### 여정 1: 사진 업로드
```
메인 → (+) 버튼 → 갤러리/카메라 선택 → 사진 선택 → 업로드 완료
```

#### 여정 2: 가족 초대
```
설정 → 가족 관리 → 초대하기 → 링크/카카오 공유 → 수락 → 가족 추가
```

#### 여정 3: 과거 사진 보기
```
타임라인 → 스크롤/달력 → 날짜 선택 → 사진 상세 → 반응/댓글
```

### 8.6 스택 선정 이유

| 선택 | 이유 |
|-----|------|
| **Rails 8** | Convention over Configuration, 빠른 MVP 개발 |
| **Turbo Native** | 웹 코드 90% 재사용, 앱스토어 배포 가능 |
| **Hotwire** | SPA 같은 UX, JavaScript 최소화 |
| **SQLite** | MVP 단순화, Solid Stack 통합, 무료 (스케일 시 PostgreSQL) |
| **Railway** | git push 배포, Volume 지원, 월 $5 (스케일 시 Kamal 2) |
| **Solid Stack** | Redis 불필요, SQLite 기반 Cache/Queue/Cable 통합 |
| **Cloudflare R2** | S3 호환, 무료 10GB, Active Storage 연동 |
