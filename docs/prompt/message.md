# 사용자 프롬프트

> 모아봄 프로젝트에서 자주 사용하는 프롬프트 템플릿 모음

---

## 0. 기능 설계 자동화

### 새 기능 설계
```text
/feature [기능명]
```

**예시:**
```text
/feature 댓글
/feature 성장기록
/feature 알림
```

**워크플로우:**
1. 폴더 생성 → `docs/features/[기능명]/`
2. 코드베이스 탐색 → 관련 모델/컨트롤러 검색
3. PRD.md 생성 → 미리보기 후 확인
4. ARCHITECTURE.md 생성 → 미리보기 후 확인
5. API_DESIGN.md 생성 → 미리보기 후 확인
6. WIREFRAME.md 생성 → 미리보기 후 확인
7. 규칙 문서 필요 여부 → 필요시 RULES.md 생성

**생성되는 폴더 구조:**
```
docs/features/[기능명]/
├── PRD.md           # 요구사항 정의
├── ARCHITECTURE.md  # 아키텍처 설계
├── API_DESIGN.md    # API 설계
├── WIREFRAME.md     # 화면 설계
└── RULES.md         # 규칙 문서 (선택)
```

## 1. 작업 관리

### 작업 리스트업
```text
@CLAUDE.md 확인하고 @plan.md 실행할 수 있도록 작업 목록 리스트업해줘
```

### 현재 진행 상황 확인
```text
@plan.md 확인하고 현재 TDD 진행 상황 요약해줘. 완료된 항목, 진행 중인 항목, 남은 항목을 정리해줘.
```

### 다음 작업 시작
```text
@plan.md에서 다음 미완료 항목 찾아서 TDD 사이클 시작해줘.
```

---

## 2. TDD 워크플로우

### RED 페이즈 (실패하는 테스트 작성)
```text
/red @plan.md의 [항목명] 테스트 작성해줘
```

### GREEN 페이즈 (테스트 통과시키기)
```text
/green 방금 작성한 테스트 통과하도록 최소 코드 구현해줘
```

### REFACTOR 페이즈 (코드 개선)
```text
/refactor 구현한 코드 정리해줘. Tidy First 원칙 적용.
```

### TDD 전체 사이클
```text
/tdd @plan.md의 [Phase X.X] 항목 TDD 사이클로 구현해줘
```

## 12. 프롬프트 작성 팁

### 컨텍스트 제공
- `@파일경로`로 관련 파일 참조
- 에러 메시지는 전체 복사해서 첨부
- 기대 동작과 실제 동작 명확히 구분

### 명확한 요청
- 한 번에 하나의 작업 요청
- 완료 조건 명시 (예: "테스트 통과할 때까지")
- 제약 조건 언급 (예: "기존 API 변경 없이")

### 참조 문서 활용
- `@docs/PRD.md` - 요구사항 확인
- `@docs/API_DESIGN.md` - API 규격
- `@docs/ARCHITECTURE.md` - 아키텍처 결정
- `@plan.md` - TDD 진행 상황
