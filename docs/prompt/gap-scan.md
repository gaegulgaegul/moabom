# 단순화/생략 작업 추적 및 반영

## 기본 사용
```text
지금까지 작업한 내용 중에 의도적으로 작업을 단순화하거나 작업하지 않은 부분을 분석해줘.

다음 카테고리로 분류해서 정리해줘:
1. **의도적 단순화**: MVP 범위로 축소한 기능
2. **임시 구현**: 나중에 개선이 필요한 코드
3. **미구현 항목**: 건너뛴 기능이나 테스트
4. **하드코딩**: 설정값이나 상수로 분리해야 할 부분
5. **에러 처리 누락**: 예외 상황 처리가 부족한 부분

각 항목에 대해:
- 현재 상태
- 개선 필요 사항
- 우선순위 (P0: 즉시, P1: 다음 스프린트, P2: 백로그)
```

## IMPLEMENTATION_GAPS.md 기반 plan.md 업데이트
```text
@docs/IMPLEMENTATION_GAPS.md 를 분석하고 @plan.md 에 반영해줘.

수행할 작업:
1. IMPLEMENTATION_GAPS.md에서 P0(즉시 반영) 항목 추출
2. plan.md에 해당 작업을 TDD 사이클로 추가
   - 각 항목을 RED/GREEN/REFACTOR 단계로 분해
   - 기존 Phase 흐름에 맞게 배치
3. P1/P2 항목은 plan.md 하단 "Future Work" 섹션에 정리

업데이트 후 변경 사항 요약해줘.
```

## 세션 종료 전 갭 분석
```text
이번 세션에서 작업한 내용을 기준으로 갭 분석해줘.

확인 사항:
1. plan.md 체크리스트 중 완료 표시했지만 실제로 불완전한 항목
2. 테스트는 통과하지만 엣지케이스 누락된 부분
3. 설계 문서(PRD, API_DESIGN, ARCHITECTURE)와 실제 구현 차이
4. TODO/FIXME 주석으로 남긴 부분

결과를 @docs/IMPLEMENTATION_GAPS.md 에 추가하고,
즉시 수정이 필요한 항목은 @plan.md 에도 반영해줘.
```

## 코드베이스 전체 갭 스캔
```text
프로젝트 전체에서 단순화/미완성 부분을 스캔해줘.

검색할 패턴:
- TODO, FIXME, HACK, XXX 주석
- raise NotImplementedError
- skip 또는 pending 테스트
- 하드코딩된 값 (매직 넘버, 문자열)
- 빈 rescue 블록
- 주석 처리된 코드

결과를 다음 형식으로 정리:
| 파일:라인 | 유형 | 내용 | 우선순위 |
|-----------|------|------|----------|

P0 항목은 @plan.md 에 즉시 추가해줘.
```
