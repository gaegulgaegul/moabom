# 디자인 시스템 프롬프트

```text
너는 모바일 앱 UX/UI를 설계하는 시니어 프로덕트 디자이너이자 프론트엔드 아키텍트다.
내가 사용 중인 디자인 시스템(모아봄 디자인 가이드)을 바탕으로, 최신 모바일 디자인 트렌드를 더 잘 반영하도록 보완 포인트를 설계해줘.

1. 현재 디자인 시스템 요약
비주얼 컨셉: “따뜻한 파스텔 + Glassmorphism”

Primary: 살구/연분홍 계열

Secondary: 민트/청록

Accent: 따뜻한 핑크

배경: 크림 톤, 텍스트: 웜 그레이

핵심 가치: 따뜻함, 단순함, 접근성, 사진 중심 레이아웃

모바일 우선 (Mobile First), 하단 탭바, Safe Area 대응

컴포넌트: 버튼, 카드, 입력, 네비게이션, Empty state 등 기본 구성 완료

상태/모션:

버튼/카드/입력에 hover/active/focus/disabled/loading 상태 정의

transition-colors / transition-all, duration 150~300ms, ease-in-out 사용

스피너, skeleton(pulse), bounce 로딩 점 등 기본 로딩 패턴 정의

접근성:

색상 대비 기준, 터치 타겟 48px, prefers-reduced-motion, ARIA 라벨 가이드 있음

다크 모드: 라이트/다크 색상 페어 정의 및 시스템 설정 연동 가이드 존재

(참고용 전체 문서는 design-system.md 로 별도 제공되어 있다고 가정해도 좋다.)

2. 목표
아래 네 가지 방향으로 트렌드에 맞는 개선안을 제안하고,
가능하면 TailwindCSS 유틸리티/컴포넌트 클래스 예시까지 포함해 달라.

콘텐츠 중심 모션 패턴 강화

사진/앨범 중심 앱이라는 특성에 맞게:

스크롤 연동 인터랙션(예: 헤더 축소/투명도 변화, 사진 그리드 페이드 인/스케일 인)

사진 상세 보기 전환(리스트 → 상세로 전환 시 scale/translate 조합)

좋아요/즐겨찾기, 댓글 작성 같은 주요 액션의 마이크로 인터랙션

각 패턴에 대해:

목적(사용자 입장에서 어떤 느낌/피드백을 주는지)

권장 애니메이션 프로퍼티(translate/scale/opacity 등)

권장 duration / easing

Tailwind 기준 클래스 예시 또는 커스텀 유틸리티 제안

온보딩 & 하이라이트 튜토리얼 패턴 정의

신규 사용자 온보딩 플로우:

2~3장 수준의 카드 또는 풀스크린 온보딩 화면 구조 제안

브랜드 톤(따뜻한 가족 사진 앱)에 맞는 카피 톤 & 레이아웃 가이드

기능 하이라이트 튜토리얼:

특정 기능(예: 첫 사진 업로드, 가족 초대)을 안내하는 모달/바텀시트/툴팁 패턴 정의

오버레이, 포커스 영역 스타일, 진입/퇴장 애니메이션 가이드

역시 Tailwind 유틸리티나 컴포넌트 클래스 형태로 예시 포함

Bottom Sheet / Modal 인터랙션 패턴

사진 업로드, 댓글 보기, 옵션 메뉴 등에 쓸 수 있는 바텀시트 패턴을:

레이아웃(핸들바, safe area, 높이 단계 – small/medium/large)

모션(슬라이드 업/다운, opacity 페이드)

제스처(스와이프 다운으로 닫기 시 애니메이션) 관점에서 설계

일반 모달(중앙 팝업)이 필요한 경우:

언제 모달, 언제 바텀시트를 써야 하는지 선택 기준

라이트/다크 모드 모두에서 어울리는 Glassmorphism/배경 오버레이 스타일

토스트/인앱 알림 패턴

성공/실패/정보 알림을 위한 상단/하단 토스트 패턴:

사용 위치(상/하), 보여지는 시간, dismiss 인터랙션

색상/아이콘/모션(살짝 위로 떠오르며 opacity in/out 등) 가이드

기존 Alert 컴포넌트와 역할을 어떻게 분리할지 정리

3. 산출물 형식
아래 형식에 맞춰 정리해줘.

섹션별 개요

각 항목(콘텐츠 중심 모션, 온보딩, 바텀시트/모달, 토스트)에 대해

“왜 필요한가 / 어떤 경험을 강화하는가”를 2~3문장으로 요약

구체 가이드

각 패턴마다:

설계 의도

시나리오 예시 (어떤 화면/행동에서 쓰일지)

모션/상태 정의 (duration, easing, 속성, 진입/퇴장)

접근성 고려사항(prefers-reduced-motion일 때 fallback 등)

코드/클래스 예시

가능하면 TailwindCSS 기준으로:

HTML 구조 예시 (간단한 div/section 수준)

class 조합 예시

필요시 @layer components 로 정의하는 컴포넌트 클래스 예시도 포함

마지막에

내가 기존 design-system.md에 어떤 섹션/하위 섹션으로 추가하면 좋은지

섹션 제목 후보(예: “10.x 콘텐츠 중심 모션 패턴”, “10.x 온보딩 & 튜토리얼”)를 제안해줘.

디자인 시스템 문서에 바로 편입 가능한 수준으로 구체적이고 실무적으로 답변해 줘.
```
