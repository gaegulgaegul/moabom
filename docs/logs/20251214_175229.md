## 주요 기능

**자동 코드 리뷰**
- PR(Pull Request) 생성 시 자동으로 코드 분석
- 버그, 보안 취약점, 성능 이슈 탐지
- 코드 스타일 및 베스트 프랙티스 제안

**통합 지원**
- GitHub, GitLab, Azure DevOps 연동
- Jira, Linear 등 이슈 트래커 연동

**특징**
- 컨텍스트 인식: 전체 코드베이스 이해 기반 리뷰
- 대화형 피드백: PR 코멘트로 AI와 대화 가능
- 커스터마이징: `.coderabbit.yaml`로 리뷰 규칙 설정

## 설정 예시

```yaml
# .coderabbit.yaml
language: ko-KR
reviews:
  profile: assertive
  path_filters:
    - "!test/**"
  auto_review:
    enabled: true
```

### 응답

## CodeRabbit은 PR 기반으로 동작합니다

main에 직접 push하면 **CodeRabbit 리뷰를 받을 수 없습니다**.

```
# CodeRabbit 동작 방식
feature-branch → PR 생성 → CodeRabbit 자동 리뷰 → main 병합
                    ↑
              여기서 리뷰 발생
```

## 권장 워크플로우

```bash
# 1. 브랜치 생성
git checkout -b feature/photo-upload

# 2. 작업 후 커밋
git add .
git commit -m "feat(photo): 사진 업로드 구현"

# 3. 브랜치 푸시
git push -u origin feature/photo-upload

# 4. GitHub에서 PR 생성 → CodeRabbit 자동 리뷰

# 5. 리뷰 반영 후 main에 병합
```

## 그래도 main에 직접 push하고 싶다면

**CodeRabbit 대신 다른 방법 고려:**

| 방법 | 설명 |
|-----|------|
| **pre-commit hook** | 푸시 전 로컬에서 검사 |
| **GitHub Actions** | 푸시 후 CI에서 분석 |
| **RuboCop** | 코드 스타일 자동 검사 |

## 추천

1인 개발이라도 **브랜치 + PR 방식**을 권장합니다:

- 코드 변경 이력이 명확해짐
- AI 리뷰로 실수 방지
- 나중에 "왜 이렇게 했지?" 추적 용이

간단한 alias 설정으로 편하게 사용 가능:

```bash
# ~/.zshrc 또는 ~/.bashrc
alias gpr='git push -u origin $(git branch --show-current) && gh pr create --fill'
```

어떤 방식으로 진행하시겠어요?

## 설정 내용

| 항목 | 설정 |
|-----|------|
| **언어** | 한국어 (ko-KR) |
| **리뷰 스타일** | assertive (적극적) |
| **자동 리뷰** | main 브랜치 대상 PR |
| **제외 파일** | lock 파일, schema.rb, credentials 등 |

## 커스텀 리뷰 지침

프로젝트 컨벤션 문서 기반으로 리뷰 기준 설정:
- Rails 컨벤션 (thin controller, N+1 방지)
- 코드 스타일 (snake_case, guard clause)
- 보안 (SQL injection, XSS)
- TDD 워크플로우

## 사용 방법

```bash
# 1. 설정 파일 커밋
git add .coderabbit.yaml
git commit -m "chore: CodeRabbit 설정 추가"
git push

# 2. coderabbit.ai에서 레포 연동 (아직 안했다면)

# 3. 브랜치 작업 후 PR 생성하면 자동 리뷰
```

## 리뷰 스타일 옵션

필요하면 `profile` 값 변경 가능:

- `chill`: 중요한 이슈만
- `assertive`: 균형 잡힌 리뷰 (현재)
- `nitpicky`: 세세한 부분까지
