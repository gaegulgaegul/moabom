### 응답

"passes"는 컨텍스트에 따라 다른 의미를 가집니다.

## 테스트 컨텍스트에서

**"test passes"** = 테스트가 통과한다 (성공)

```ruby
# 테스트가 pass하면 = 예상대로 동작함
test "should be valid" do
  assert photo.valid?  # true면 pass
end
```

- **pass** (통과): 테스트 성공 ✅
- **fail** (실패): 테스트 실패 ❌

## TDD에서

프로젝트의 TDD 워크플로우:
1. **RED**: 실패하는 테스트 작성 (fails)
2. **GREEN**: 테스트가 통과하도록 구현 (passes)
3. **REFACTOR**: 코드 개선

## Rails에서 다른 의미

```ruby
# Strong Parameters에서 "permit"과 함께 사용
params.permit(:name)  # 허용된 파라미터만 통과(pass)시킴
```

```
이건 Claude Code의 레퍼럴/초대 시스템과 관련된 내장 명령어입니다. 제가 SlashCommand 도구로 실행할 수 있는 커스텀 명령어가 아니라, Claude Code 애플리케이션 자체의 기능입니다.
```

</details>

### 응답

아, 그건 **Claude Code CLI의 내장 명령어**입니다!

## `/passes`

**친구에게 Claude Code 무료 1주일 이용권을 공유**하는 기능입니다.

- Claude Code 유료 구독자가 친구를 초대할 때 사용
- 초대받은 사람은 1주일간 무료로 Claude Code 사용 가능
- 일종의 레퍼럴/초대 시스템

이건 제가 실행하는 커스텀 슬래시 명령어(`/sc:*`, `/tdd` 등)가 아니라, Claude Code 앱 자체에 내장된 기능이라서 제 Available Commands 목록에는 없었습니다.

터미널에서 직접 `/passes`를 입력하시면 사용하실 수 있습니다.
